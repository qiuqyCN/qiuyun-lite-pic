# 秋云轻图 - 微信小程序需求文档（纯前端 + 广告变现版）

---

## 一、小程序基本信息

| 项目 | 内容 |
|------|------|
| **小程序名称** | 秋云轻图 |
| **英文名称** | QiuYun LitePic |
| **Slogan** | 轻点之间，美图呈现 |
| **服务类目** | 工具-图片处理 |
| **主体类型** | 个人开发者 |
| **技术方案** | 纯前端实现，零后端依赖 |

### 1.1 名称设计理念
- **秋云**：取自"秋高气爽，云淡风轻"，传达轻盈、简洁的产品气质
- **轻图**：强调轻量级图片处理，突出产品核心功能

### 1.2 小程序描述
> 秋云轻图是一款完全免费的轻量级图片处理工具，所有功能均在本地完成，无需上传服务器，保护隐私安全。提供图片压缩、格式转换、尺寸调整、滤镜美化等实用功能。

## 二、技术架构（纯前端方案）

### 2.1 核心技术栈
| 层级 | 技术 | 说明 |
|------|------|------|
| 框架 | 微信小程序原生 | 稳定可靠 |
| 语言 | TypeScript | 类型安全 |
| 图片处理 | Canvas 2D API | 本地处理，不上传 |
| 存储 | wx.getFileSystemManager | 本地临时文件 |
| 状态管理 | 全局状态 + Storage | 无需后端数据库 |

### 2.2 为什么选择纯前端？

#### ✅ 优势
| 优势 | 说明 |
|------|------|
| **零成本** | 无需服务器、域名、SSL证书 |
| **隐私安全** | 图片本地处理，不上传云端 |
| **速度快** | 无需网络传输，即时处理 |
| **易维护** | 无需部署后端，更新简单 |
| **合规简单** | 无需ICP备案、隐私政策简单 |

#### ⚠️ 限制与解决方案
| 限制 | 解决方案 |
|------|----------|
| 无法做AI抠图 | 使用传统图像算法（色域分割） |
| 无法云端存储历史 | 本地Storage存储+导出到相册 |
| 大图片可能卡顿 | 分块处理+进度提示 |
| 无法跨设备同步 | 引导用户保存到相册/微信收藏 |

### 2.3 图片处理技术方案

```
用户选择图片
    ↓
wx.getImageInfo 获取信息
    ↓
Canvas 绘制处理
    ↓
canvas.toTempFilePath 导出
    ↓
本地保存/分享
```

## 三、功能需求规划

### 3.1 V1.0 核心功能（纯前端可实现）

#### 3.1.1 图片压缩 ⭐核心
- **智能压缩**：根据目标大小自动计算压缩比例
- **自定义压缩**：滑动条调节质量（10%-100%）
- **目标大小模式**：输入目标KB数，自动优化
- **预览对比**：左右滑动对比原图/压缩后
- **批量处理**：支持最多9张（受小程序限制）

**技术实现**：`canvas.drawImage` + `canvas.toTempFilePath({ quality })`

#### 3.1.2 尺寸调整 ⭐核心
- **常用比例**：1:1、4:3、3:4、16:9、9:16
- **社交平台预设**：
  - 微信朋友圈封面：1280×720
  - 抖音视频封面：1080×1920
  - 小红书封面：1242×1660
  - 头像：200×200、500×500
- **证件照尺寸**：
  - 一寸：295×413px
  - 二寸：413×626px
  - 小二寸：413×531px
- **自由裁剪**：手势缩放+拖动

**技术实现**：Canvas裁剪 + `clip` 方法

#### 3.1.3 格式转换 ⭐核心
- **支持格式**：JPG ↔ PNG ↔ WEBP
- **透明处理**：PNG转JPG时自动填充白色/自定义背景色
- **批量转换**：一键转换多张

**技术实现**：`canvas.toTempFilePath({ fileType })`

#### 3.1.4 基础滤镜 ⭐核心
- **预设滤镜**（15款）：
  - 原图、明亮、复古、黑白、胶片、清新、美食、人像、风景、夜景、暖色、冷色、高饱和、低饱和、柔光
- **参数调节**：
  - 亮度：-100 ~ +100
  - 对比度：-100 ~ +100
  - 饱和度：-100 ~ +100
  - 色温：-100 ~ +100
  - 锐化：0 ~ 100
- **一键优化**：自动亮度+对比度调整

**技术实现**：Canvas像素操作 `getImageData` + 滤镜算法

### 3.2 V2.0 进阶功能（纯前端算法）

#### 3.2.1 简单抠图（色域分割算法）
- **纯色背景抠图**：识别纯色背景（白/蓝/红/绿）并移除
- **手动擦除**：画笔涂抹去除/保留区域
- **边缘平滑**：简单羽化处理

**技术实现**：颜色阈值判断 + 像素替换

- **背景颜色**：纯色/渐变背景

#### 3.2.3 水印功能

  - 自定义文字内容

#### 3.2.4 图片标注

- **画笔**：自由涂鸦
- **文字标注**：添加说明文字

## 四、广告变现设计

### 4.1 广告类型与场景

| 广告类型 | 使用场景 | 展示频率 |
|----------|----------|----------|
| **激励视频广告** | 解锁高级功能/去水印/批量处理 | 用户主动触发 |
| **插屏广告** | 处理完成保存后 | 每3次操作展示1次 |

| **格子广告** | 首页功能入口下方 | 常驻 |

### 4.2 广告场景详细设计

#### 场景1：激励视频广告（主要收入来源）

| 触发点 | 奖励内容 | 限制 |
|--------|----------|------|
| 点击"批量处理" | 解锁本次批量处理 | 每日限3次 |
| 点击"高清导出" | 本次导出原画质 | 无限制 |
| 点击"去广告24小时" | 24小时内无广告 | 每日限1次 |
| 点击"高级滤镜" | 解锁全部滤镜1小时 | 无限制 |
| 处理次数用完 | 获得3次额外次数 | 每日限5次 |

#### 场景2：插屏广告
- **触发时机**：用户完成图片处理并保存后
- **频率控制**：每成功保存3张图片展示1次
- **避免打扰**：处理过程中不展示，保存失败不展示

#### 场景3：Banner广告
- **位置**：处理页面底部、首页底部
- **样式**：自适应宽度，高度固定

## 八、核心算法伪代码

### 8.1 图片压缩算法
```typescript
async function compressImage(
  src: string, 
  quality: number,
  targetSize?: number // KB
): Promise<string> {
  // 1. 获取图片信息
  const info = await wx.getImageInfo({ src });
  
  // 2. 创建Canvas
  const ctx = wx.createCanvasContext('compressCanvas');
  
  // 3. 绘制图片
  ctx.drawImage(src, 0, 0, info.width, info.height);
  await ctx.draw();
  
  // 4. 如果指定了目标大小，二分查找最佳质量
  let finalQuality = quality;
  if (targetSize) {
    finalQuality = await findBestQuality(src, targetSize);
  }
  
  // 5. 导出压缩后的图片
  const result = await wx.canvasToTempFilePath({
    canvasId: 'compressCanvas',
    quality: finalQuality / 100,
    fileType: 'jpg'
  });
  
  return result.tempFilePath;
}
```

### 8.2 滤镜算法（亮度调节示例）
```typescript
function applyBrightness(
  imageData: ImageData, 
  value: number
): ImageData {
  const data = imageData.data;
  const adjustment = value * 2.55; // -255 ~ 255
  
  for (let i = 0; i < data.length; i += 4) {
    data[i] = Math.min(255, Math.max(0, data[i] + adjustment));     // R
    data[i + 1] = Math.min(255, Math.max(0, data[i + 1] + adjustment)); // G
    data[i + 2] = Math.min(255, Math.max(0, data[i + 2] + adjustment)); // B
    // data[i + 3] 是Alpha通道，保持不变
  }
  
  return imageData;
}
```

## 九、风险评估

### 9.1 技术风险
| 风险 | 影响 | 解决方案 |
|------|------|----------|
| 大图片处理卡顿 | 用户体验差 | 分块处理+进度条+异步处理 |
| Canvas内存溢出 | 小程序崩溃 | 及时释放Canvas，控制并发 |
| 滤镜算法性能差 | 预览卡顿 | 使用WebGL或降低预览分辨率 |

### 9.2 商业风险
| 风险 | 影响 | 解决方案 |
|------|------|----------|
| 广告eCPM低 | 收入少 | 多场景广告+优化点击率 |
| 用户反感广告 | 流失率高 | 激励视频为主，插屏控制频率 |
| 竞品免费无广告 | 竞争力弱 | 强调隐私安全（本地处理）作为卖点 |

### 9.3 合规风险
| 风险 | 影响 | 解决方案 |
|------|------|----------|
| 广告审核不通过 | 无法上线 | 严格遵守微信广告规范 |
| 用户投诉广告多 | 下架风险 | 提供"去广告"选项（看广告解锁） |
